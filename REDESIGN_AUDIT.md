## Status

Implemented and verified for the consumer app. The redesign keeps the existing app architecture while applying the approved "Guided Journey Path" direction across auth entry, onboarding profile steps, booking, and community tabs.

## Evidence

- Expo SDK 56 docs were checked before code changes, per `AGENTS.md`.
- TypeScript passes with `npx tsc --noEmit --pretty false`.
- Focused flow tests pass: auth screens, onboarding flow, booking screens/components/copy, screen layout, and navigation lifecycle.
- Mobile web screenshot captured at `output/playwright/authentic-welcome-mobile.png`.

## Requirement Mapping

- Theme additions: implemented in `src/theme/colors.ts`, `spacing.ts`, `typography.ts`, `gradients.ts`, `motion.ts`, and `shadows.ts`.
- Gradients: `GradientHero` uses `expo-linear-gradient`; `ProgressBar` and `JourneyPath` use the path-fill gradient.
- Layered depth: `Card` has `elevated?: boolean`, shared shadows exist in `src/theme/shadows.ts`, and selected controls use gold glow styling.
- Symbolic icon system: `SymbolicIcon` uses Feather outline icons for thematic icons; Ionicons remain for utility icons and a test fallback.
- Journey path: `JourneyPath` supports 9-node paths, active/completed/upcoming states, animated fill, active-node pulse, and compact progress-bar fallback below narrow widths.
- Motion: button press, choice-card press, section entrances, symbolic icon pulse, ambient hero drift, time-slot entrance, journey-path advance, and confirmation shimmer are implemented. `useReduceMotion` collapses motion when OS reduce-motion is enabled.
- Photography: no stock/decorative photography was introduced. Guide/member photos remain the only profile imagery.
- Welcome: dark gradient hero, journey path, eyebrow, Playfair headline, supportive copy, and symbolic fellowship icon are implemented.
- Onboarding profile steps: eyebrow/header/body structure, symbolic icons where appropriate, no icon on Profile Basics, choice cards, premium inputs, and staggered entrances are implemented.
- Schedule: journey path step, guide card, date picker, animated time-slot section, and selected slot glow are implemented.
- Confirmation: journey path step, dark hero, rose-gold warmth, gold checkmark circle, and one-time shimmer are implemented.
- Home: sand gradient hero plus elevated "Next Step" card with thin gold left border are implemented.
- Events: flat scan-focused cards with no gradients are implemented.
- Foundations: sand gradient progress card with gradient progress fill is implemented.
- Profile: no gradients/glow; calm utilitarian styling is preserved.

## Intentional Divergence

The design reference describes a 9-screen conceptual onboarding journey. The existing app persists seven onboarding profile data steps and keeps schedule/confirmation in the booking navigator. The implementation maps the 9-step journey across existing navigation boundaries instead of rewriting persistence, validation, and recovery flows. This is documented in `DECISIONS.md`.
